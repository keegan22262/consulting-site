We are now designing the full Insights page architecture for the Rill Singh Limited website.

The Insights page will serve as the firm's thought leadership platform.

The page must combine the structural logic used by McKinsey with the institutional tone used by BlackRock.

The objective is to make the page feel like an institutional research platform rather than a blog.

The design must support the current 15 insights and scale easily to 50+ insights in the future.

Each insight already exists in the CMS and contains:

• headline
• meaning
• what RSL does
• source link

Each insight card must link to its own detailed insight page using the existing links.

Do not remove or modify the links. Preserve them exactly as they exist.

---

PAGE STRUCTURE

The Insights page must follow this structure.

1. Insights Hero
2. Featured Insight
3. Topic Filters
4. Insights Grid
5. Featured Analysis Section
6. Insights Carousel
7. Research / Newsletter CTA

---

SECTION 1 — INSIGHTS HERO

Create a large editorial hero introducing the Insights platform.

Hero title example:

Insights & Perspectives

Supporting text example:

Strategic analysis on the forces shaping institutions, markets, and transformation across Africa.

Design requirements:

• large editorial typography
• calm institutional tone
• cinematic background image with navy overlay
• strong whitespace and balance

This section should feel authoritative and calm.

---

SECTION 2 — FEATURED INSIGHT

Display one dominant insight card below the hero.

This should visually highlight a flagship insight.

Card structure:

• large cinematic image
• category label
• insight headline
• short description
• "Read Full Insight →"

Text should appear toward the lower-left of the card.

Use a strong navy gradient overlay to maintain readability.

SECTION 4 — INSIGHTS GRID

Display the insights as a structured editorial grid.

Grid layout:

Desktop: 3 columns
Tablet: 2 columns
Mobile: 1 column

Each insight card must contain:

• editorial background image
• category label
• insight headline
• short description (1–2 lines)
• publication metadata
• link to the detailed insight page

Metadata format example:

RSL Perspectives • November 2025

Card interaction:

On hover:

• background image darkens slightly
• headline becomes more dominant
• "Read Insight →" appears

This grid must support up to 50+ insights in the future.

Pagination or load-more functionality should be considered in the design.

DESIGN RULES

The Insights page must follow these design principles:

1. It must feel like institutional research rather than a marketing blog.
2. Typography should be large, calm, and authoritative.
3. Images must be cinematic and editorial.
4. Maintain the navy color palette used across the site.
5. Maintain the rounded-card system used across the homepage.
6. Maintain consistent spacing and layout rhythm.

---

THOUGHT LEADERSHIP ENGINE STRUCTURE

The page must follow the same intellectual architecture used by McKinsey and BCG to create thought leadership authority.

This means the page must communicate:

Strategic context
Research insights
Institutional expertise
Practical implementation perspective

Every insight should show:

• what the trend means
• how institutions should respond
• how RSL supports execution

This approach ensures the insights platform becomes a strategic authority engine rather than a content list.

---

IMPORTANT NAVIGATION RULE

Every insight card must remain clickable and link directly to the detailed insight page using the existing links.

These links must be preserved exactly as provided.

The detailed insight page layout will be designed separately.

---

FINAL OBJECTIVE

The Insights page should feel like a strategic research platform similar to those used by leading global consulting and investment institutions.

Visitors should immediately understand that the firm produces institutional-grade insights and advisory thinking.

The page must support growth from 15 insights today to 50+ insights in the future without requiring structural redesign.
